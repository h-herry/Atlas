## 1. 项目架构 / Project Architecture

**整体架构 / Overall Architecture**：Spring Boot 3.2 + Spring Cloud 2023 + JDK 17

基于 Spring Cloud 微服务体系：Nacos 管理服务注册与配置，Gateway 作为统一入口，共 10 个 Maven 模块。分布式事务走 Seata AT 模式，ShardingSphere 5.4.1 做分库分表，Sentinel 控熔断降级，Zipkin 覆盖全链路追踪。

---

Adopts Spring Cloud microservice architecture with Nacos for service registration and configuration management, Gateway as unified entry point, totaling 10 Maven modules. Distributed transactions are coordinated via Seata AT mode, ShardingSphere 5.4.1 handles database/table sharding, Sentinel provides circuit breaking and rate limiting, and Zipkin enables full distributed tracing.

## 4. 安全设计 / Security Design

### 4.1 JWT 无状态认证 / JWT Stateless Authentication

```
请求 / Request → JwtAuthenticationFilter → TokenService 验证 / Verification → SecurityContext → Controller
```

- Access Token 有效期 2 小时，存储于 Redis / Access Token: 2h validity, stored in Redis
- Refresh Token 用于无感刷新 / Refresh Token for seamless renewal
- TokenService 缓存用户权限列表，减少数据库查询 / TokenService caches permissions to reduce DB queries
- 登录失败锁定：连续失败 5 次锁定 15 分钟 / Login lockout: 5 consecutive failures → 15 min lockout

### 4.2 数据权限模型 / Data Permission Model

`DataScopeInterceptor` 通过 AOP 切面读取 `role.data_scope` 和 `user.dept_id`，自动改写 SQL 的 WHERE 条件 / Reads role.data_scope and user.dept_id via AOP, auto-rewrites SQL WHERE clauses：

| 级别 / Level | 值 / Value | 含义 / Meaning | WHERE 条件 / WHERE Clause |
|------|-----|------|------------|
| 仅本人 / Self | 1 | 只能看自己的数据 / Own data only | `created_by = current_user_id` |
| 本部门 / Dept | 2 | 只能看本部门数据 / Own department only | `dept_id = current_dept_id` |
| 本部门及子部门 / Dept & Sub | 3 | 含所有下级部门 / Including all sub-departments | `dept_path LIKE '/1/3%'` |
| 全部 / All | 4 | 全部数据 / All data | 无限制 / No restriction |

### 4.3 开放平台鉴权 / Open Platform Authentication

- AppKey + AppSecret + HMAC-SHA256 签名验证 / HMAC-SHA256 signature verification
- 防重放攻击：±5min 时间窗口（timestamp + nonce） / Anti-replay: ±5min window
- 客户端级限流：`rate_limit_per_min` 可配置 / Client-level rate limiting: configurable
- API 调用日志：`api_call_log` 全量记录（路径/方法/IP/状态码/耗时） / Full call logging: path, method, IP, status, duration

## 5. 合同状态机 / Contract State Machine

### 5.1 状态定义 / State Definitions（ContractStatusEnum）

| Code | 状态 / State | 说明 / Description |
|------|------|------|
| 0 | DRAFT（草稿） | 新建合同，可编辑 / New contract, editable |
| 1 | SUBMITTED（已提交） | 提交审批 / Submitted for approval |
| 2 | REVIEWING（审核中） | 审批进行中 / Under review |
| 3 | APPROVED（审核通过） | 审批通过 / Approved |
| 4 | REJECTED（驳回） | 驳回，可重新提交 / Rejected, can resubmit |
| 5 | SIGNED（已签署） | 双方签署完成 / Both parties signed |
| 6 | EXECUTING（执行中） | 合同执行中 / In execution |
| 7 | CHANGING（变更中） | 合同变更流程中 / Under change |
| 8 | TERMINATED（已终止） | 终态 / Terminal state |
| 9 | COMPLETED（已完成） | 终态 / Terminal state |

### 5.2 流转规则 / Transition Rules

- **入口守卫 / Entry Guard**：`ContractStatusEnum.validateTransition(fromCode, toCode)` 校验流转合法性 / Validates transition legality
- **allowTargets**：每个状态枚举内置合法目标状态集合 / Each state enum has built-in legal target set
- 8（已终止）和 9（已完成）为终态，不可再流转 / 8 and 9 are terminal, cannot transition further
