## 1. 项目概述 / Project Overview

### 1.1 项目定位 / Positioning

Atlas SRM（Supplier Relationship Management）是企业级供应商关系管理系统，覆盖供应商全生命周期管理，从准入注册、资质审核、绩效评估到协同交付、风险监控、合同签署、采购执行与结算对账，形成完整的供应商管理闭环。

### 1.2 版本信息 / Version Info

| 项 | 值 |
|---|---|
| 版本号 | 1.2.502 |
| JDK | 17 |
| Spring Boot | 3.2.5 |
| Spring Cloud | 2023.0.1 |
| Spring Cloud Alibaba | 2023.0.1.0 |
| MyBatis-Plus | 3.5.6 |
| Flowable | 7.0.1 |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis（Redisson 3.27.2） |
| 消息队列 | RocketMQ 5.1.4 |

### 1.3 业务范围 / Business Scope

覆盖供应商全生命周期 10 大业务域：

1. 供应商准入与注册 / Supplier Onboarding & Registration
2. 供应商资质与分类 / Supplier Qualification & Classification
3. 供应商绩效评估 / Supplier Performance Evaluation
4. 供应商协同门户 / Supplier Collaboration Portal
5. 采购寻源与执行 / Procurement Sourcing & Execution
6. 合同管理与电子签署 / Contract Management & E-Signature
7. 订单与交付管理 / Order & Delivery Management
8. 收货质检与三单匹配 / Receipt, Quality & Three-Way Match
9. 结算对账与付款 / Settlement, Reconciliation & Payment
10. 风险监控与预警 / Risk Monitoring & Alert

## 3. 模块详细设计 / Module Detailed Design

### 3.1 atlas-common — 公共基础设施 / Common Infrastructure

**定位**：非独立运行服务，为所有业务模块提供共享的基础设施层。承载安全组件（JWT 过滤器 / RBAC 拦截器 / 权限切面）、消息中心（WebSocket 推送 / 邮件 / 短信）、审计日志、国际化、缓存配置等公共能力。

**包结构**：
```
com.atlas.common/
├── aspect/            # 审计日志切面、限流切面
├── async/             # 异步任务配置
├── cache/config/      # Redis / Cache 配置
├── config/            # Sentinel 配置
├── core/              # Result、ErrorCode、BizException
├── entity/            # 公共 Entity（RBAC 实体、审计日志、组织架构）
├── export/            # 导出模板
├── feign/             # Feign 拦截器（TraceId 传递）
├── i18n/              # 国际化
├── mapper/            # 公共 Mapper（RBAC / 审计 / 国际化）
├── message/           # 消息中心（WebSocket / 邮件 / 短信 / 死信）
├── security/          # JWT 过滤器 / 数据权限拦截器 / 权限切面
├── template/          # 审批模板
├── websocket/         # WebSocket 鉴权拦截器
└── web/filter/        # TraceId 过滤器
```

**核心 Entity（位于 common）**：

|------|------|------|
| SysPermission | sys_permission | RBAC 权限定义 |
| SysRole | sys_role | RBAC 角色定义 |
| SysRolePermission | sys_role_permission | 角色-权限关联 |
| SysUserRole | sys_user_role | 用户-角色关联 |

### 3.2 atlas-gateway — API 网关 / API Gateway

**端口**：8080

**定位**：系统统一入口，负责请求路由、JWT 鉴权、Sentinel 限流降级、跨域处理。

**包结构**：
```
com.atlas.gateway/
├── GatewayApplication.java
├── config/            # Gateway 路由配置、跨域配置
└── filter/            # 全局过滤器（鉴权、日志）
```

**核心能力**：
- Spring Cloud Gateway 路由转发到后端 11 个微服务
- 全局 JWT Token 校验过滤器，非法请求在网关层直接拦截
- Sentinel 网关限流，保护后端服务
- 请求日志记录

**路由规则**：
| 路由前缀 | 目标服务 | 说明 |
|------|------|------|
| `/auth/**` | atlas-user:8081 | 认证接口（登录/登出/刷新 Token） |
| `/user/**` | atlas-user:8081 | 用户管理 |
| `/api/system/**` | atlas-user:8081 | RBAC 权限管理 |
| `/supplier/**` | atlas-supplier:8082 | 供应商管理 |
| `/workflow/**` | atlas-workflow:8083 | 工作流 |
| `/purchase/**` | atlas-purchase:8084 | 采购管理 |
| `/inventory/**` | atlas-inventory:8085 | 库存管理 |
| `/contract/**` | atlas-contract:8086 | 合同管理 |
| `/material/**` | atlas-material:8088 | 物料管理 |
| `/receipt/**` | atlas-receipt:8089 | 收货结算 |
| `/quality/**` | atlas-quality:8091 | 质量管理 |
| `/open/**` | atlas-open:8098 | 开放平台 |

## 6. 安全设计 / Security Design

### 6.1 双通道 JWT 认证 / Dual-Channel JWT Authentication

Atlas 支持企业端（enterprise）和供应商端（supplier）两个独立通道，共用同一套 JWT 基础设施但通过 `channel` 字段区分。

```
POST /auth/login
Body: { "username": "...", "password": "...", "channel": "enterprise|supplier" }
     │
     ▼
AuthService.login()
     ├─ 验证凭据（BCrypt 密码比对）
     ├─ 登录失败次数检查（Redis 计数，5 次锁定 30 分钟）
     ├─ 查询用户角色 + 权限列表
     ├─ 生成 JWT Token
     │   ├─ Header: { "alg": "HS512" }
     │   └─ Payload: {
     │         "sub": userId,
     │         "username": "...",
     │         "channel": "enterprise|supplier",
     │         "supplierId": "..." (供应商通道专属),
     │         "roles": [...],
     │         "permissions": [...],
     │         "iat": ...,
     │         "exp": ...
     │      }
     ├─ 记录登录日志 (LoginLog)
     └─ 返回 { accessToken, refreshToken, userId, ... }
```

**Token 生命周期**：
- AccessToken：2 小时有效
- RefreshToken：7 天有效，用于无感刷新

### 6.2 RBAC 权限模型 / RBAC Permission Model

```
                 ┌──────────────┐
                 │   User       │
                 │   用户       │
                 └──────┬───────┘
                        │ N:M (sys_user_role)
                 ┌──────▼───────┐
                 │   Role       │──── 六级数据范围 / Data Scope
                 │   角色       │     ├─ SCOPE_ALL (全部)
                 └──────┬───────┘     ├─ SCOPE_DEPT (本部门)
                        │ N:M         ├─ SCOPE_DEPT_AND_CHILD (本部门及下级)
                 ┌──────▼───────┐     ├─ SCOPE_SELF (仅本人)
                 │  Permission  │     ├─ SCOPE_SUPPLIER (供应商级)
                 │   权限       │     └─ SCOPE_CUSTOM (自定义)
                 └──────────────┘
```

- **RBAC 实体**位于 `atlas-common`（`SysPermission` / `SysRole` / `SysRolePermission` / `SysUserRole`）
- **RBAC 控制器**位于 `atlas-user` 的 `com.atlas.user.system.controller` 子包
- **数据权限**通过 MyBatis `DataScopeInterceptor` 注入 SQL 条件实现行级数据隔离
- **权限校验**通过 `@RequirePermission` 注解 + `PermissionAspect` 切面实现方法级拦截

### 6.3 数据隔离 / Data Isolation

供应商门户端用户仅能访问自身供应商数据。隔离通过两个层面实现：

1. **JWT Payload 层**：供应商通道 Token 含 `supplierId`，网关和 SecurityContext 中传递
2. **SQL 层**：`DataScopeInterceptor` 自动在查询条件中加入 `supplier_id = ?` 或 `scope_dept_id IN (...)`

### 6.4 Token 黑名单 / Token Blacklist

登出时将 AccessToken 加入 Redis 黑名单（Key: `token:blacklist:<token_hash>`，TTL = Token 剩余有效期）。网关层验证 Token 时先查黑名单，命中则直接拒绝。

### 6.5 防重放攻击 / Anti-Replay

通过 Nonce + Timestamp 机制：客户端请求携带一次性 Nonce（UUID），服务端 Redis 记录 Nonce（TTL 5 分钟），重复 Nonce 的请求直接拒绝。

### 6.6 敏感数据脱敏 / Data Masking

`@Sensitive` 注解标记敏感字段（手机号、身份证、银行卡），序列化时自动脱敏。脱敏规则内置于 common 的 JSON 序列化配置中。

## 7. 接口规范 / API Specification

### 7.1 统一响应格式 / Unified Response Format

```json
// 成功 / Success
{
  "code": 200,
  "message": "操作成功",
  "data": { ... },
  "timestamp": 1718764800000
}

// 分页 / Pagination
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "records": [ ... ],
    "total": 150,
    "pageNum": 1,
    "pageSize": 20,
    "pages": 8
  },
  "timestamp": 1718764800000
}

// 失败 / Error
{
  "code": 40001,
  "message": "用户名或密码错误",
  "data": null,
  "timestamp": 1718764800000
}
```

`Result<T>` 和 `PageResult<T>` 位于 `com.atlas.common.core`。

### 7.2 分页规范 / Pagination Convention

| 参数 | 类型 | 默认值 | 说明 |
|------|------|------|------|
| `pageNum` | int | 1 | 页码（1-based） |
| `pageSize` | int | 20 | 每页条数（最大 100） |
| `sortField` | String | — | 排序字段 |
| `sortOrder` | String | desc | asc / desc |

### 7.3 异常处理体系 / Exception Handling

```
BizException（业务异常，ErrorCode 枚举码）
    ├─ 40001: 用户名或密码错误
    ├─ 40002: Token 已过期
    ├─ 40003: 权限不足
    ├─ 40004: 数据不存在
    ├─ 40005: 重复提交
    └─ ...

全局异常处理器 @RestControllerAdvice
    ├─ BizException → Result(code, message)
    ├─ MethodArgumentNotValidException → Result(400, 参数校验失败详情)
    ├─ AccessDeniedException → Result(403, 权限不足)
    └─ Exception → Result(500, 系统内部错误)
```
