## 7. Seata 分布式事务 / Seata Distributed Transaction

### 7.1 Seata Server 部署 / Seata Server Deployment

Seata Server 1.8.0 通过 Docker Compose 启动，注册到 Nacos（`SEATA_GROUP`） / Started via Docker Compose, registered with Nacos (SEATA_GROUP):

```yaml
seata-server:
  image: seataio/seata-server:1.8.0
  ports:
    - "8091:8091"
  environment:
    - SEATA_IP=host.docker.internal
    - SEATA_PORT=8091
```

### 7.2 undo_log 表 / undo_log Table

- `atlas_inventory.undo_log`: V5 迁移脚本创建 / Created by V5 migration
- 全局 `undo_log` / Global `undo_log`: V93 迁移脚本创建 / Created by V93 migration

### 7.3 应用配置 / Application Configuration

各模块 `application.yml` / Each module's `application.yml`:

```yaml
seata:
  tx-service-group: atlas-tx-group
  registry:
    type: nacos
    nacos:
      server-addr: ${NACOS_SERVER:localhost:8848}

      server-addr: ${NACOS_SERVER:localhost:8848}
```

### 8.1 atlas-purchase 分片策略 / atlas-purchase Shard Strategy

| 表 / Table | 分库策略 / DB Strategy | 分表策略 / Table Strategy | 分片数 / Shards |
|------|------|------|:---:|
| purchase_order | `ds$→{id % 2}` | `purchase_order_$→{id % 4}` | 2库 × 4表 = 8 分片 / 2 DBs × 4 tables = 8 shards |
| audit_log | `ds$→{month % 2}` | `audit_log_$→{month % 4}` | 2库 × 4表 = 8 分片 / 2 DBs × 4 tables = 8 shards |

### 8.2 atlas-open 分片策略 / atlas-open Shard Strategy

| 表 / Table | 分库策略 / DB Strategy | 分表策略 / Table Strategy | 分片数 / Shards |
|------|------|------|:---:|
| api_integration_log | `ds$→{config_id % 2}` | `api_integration_log_$→{config_id % 4}` | 2库 × 4表 = 8 分片 / 2 DBs × 4 tables = 8 shards |

### 8.3 配置示例 / Configuration Example

```yaml
spring:
  shardingsphere:
    mode:
      type: Standalone
      repository:
        type: JDBC
    datasource:
      names: ds0,ds1
      # ... 数据源配置 / Data source config ...
    rules:
      sharding:
        tables:
          purchase_order:
            actual-data-nodes: ds$->{0..1}.purchase_order_$->{0..3}
            database-strategy:
              standard:
                sharding-column: id
                sharding-algorithm-name: order-db-inline
            table-strategy:
              standard:
                sharding-column: id
                sharding-algorithm-name: order-table-inline
```

## 9. Sentinel 熔断降级 / Sentinel Circuit Breaking & Rate Limiting

### 9.1 Dashboard 接入 / Dashboard Access

Sentinel Dashboard 通过 Docker 启动，端口 8080。各模块 `application.yml` / Sentinel Dashboard started via Docker, port 8080. Each module's `application.yml`:

```yaml
spring:
  cloud:
    sentinel:
      transport:
        dashboard: ${SENTINEL_DASHBOARD:localhost:8080}
        port: 8719
      datasource:
        ds1:
          nacos:
            server-addr: ${NACOS_SERVER:localhost:8848}
            namespace: ${NACOS_NAMESPACE:atlas}
            data-id: ${spring.application.name}-flow-rules
            group-id: SENTINEL_GROUP
            data-type: json
            rule-type: flow
        ds2:
          nacos:
            server-addr: ${NACOS_SERVER:localhost:8848}
            namespace: ${NACOS_NAMESPACE:atlas}
            data-id: ${spring.application.name}-degrade-rules
            group-id: SENTINEL_GROUP
            data-type: json
            rule-type: degrade
feign:
  sentinel:
    enabled: true
```

### 9.2 规则持久化 / Rule Persistence

流控与降级规则持久化到 Nacos SENTINEL_GROUP，Dashboard 重启不丢失。已为 atlas-open 和 atlas-purchase 配置降级规则 JSON。 / Flow and degrade rules persisted to Nacos SENTINEL_GROUP, survive Dashboard restarts. Degrade rule JSON already configured for atlas-open and atlas-purchase.
