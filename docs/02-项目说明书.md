## 1. 项目定位 / Project Positioning

**Atlas 企业采购管理系统（SRM + 采购）** 是一个面向供应链行业的 Java 全栈学习项目，同时也是企业级微服务快速开发参考架构。

项目以"供应商关系管理 → 采购管理 → 合同签订 → 收货质检 → 库存管理"为业务主线，覆盖 SRM 四大域、9 种采购模式、物料管理全生命周期，集成 RBAC 权限、工作流审批、分布式事务、分库分表、限流熔断、链路追踪、审计日志等企业级基础设施，并提供标准化开放平台供 ERP/OA/WMS 等外部系统对接。

---

**Atlas Enterprise Procurement Management System (SRM + Procurement)** is a full-stack Java learning project for the supply chain industry, also serving as an enterprise microservice rapid development reference architecture.

Following the business chain "Supplier Relationship Management → Procurement → Contract → Receipt QC → Inventory", it covers SRM's four domains, 9 procurement modes, and the full material management lifecycle, integrating RBAC permissions, workflow approval, distributed transactions, database/table sharding, rate limiting & circuit breaking, distributed tracing, audit logging, and a standardized open platform for ERP/OA/WMS integration.

- **项目版本 / Project Version**：v1.2.21
- **GroupId / ArtifactId**：`com.atlas` / `atlas`
- **Java 版本 / Java Version**：JDK 17

- **模块总数 / Total Modules**：10
- **Java 文件 / Java Files**：291
- **数据库表 / Database Tables**：43 张（SQL 声明） / 43 tables (SQL-declared) + Flowable engine tables
- **SQL 迁移脚本 / SQL Migrations**：22 个（V1 ~ V93）

## 2. 功能模块 / Functional Modules

### 2.1 用户权限 / User & Permissions

- **RBAC 模型 / RBAC Model**：用户 → 角色 → 权限，支持菜单/按钮/接口三级权限粒度 / User → Role → Permission, 3-tier granularity (menu/button/API)
- **部门树 / Department Tree**：`dept_path` 字段实现层级关系，支持无限级部门嵌套 / dept_path for hierarchy, unlimited nesting
- **JWT 双 Token / JWT Dual Token**：Access Token（2h）+ Refresh Token，无状态认证 / Stateless authentication
- **数据权限 / Data Scope**：4 级数据隔离（仅本人 / 本部门 / 本部门及子部门 / 全部） / 4-level isolation
- **登录安全 / Login Security**：连续失败锁定 15 分钟、Token 黑名单登出失效 / 15 min lockout, token blacklist logout

### 2.2 供应商关系管理 SRM / Supplier Relationship Management

供应商管理模块涵盖 SRM 核心功能域，对标某行业头部厂商四域能力： / Covers SRM core domains, benchmarked against industry standards:

**2.2.1 准入管理 / Access Management**

- 发布招募公告（按品类、资质要求），供应商在线注册 / Publish recruitment notices, online supplier registration
- 三级审批流程：初审 → 现场考察 → 终审，审批流转含状态机 / 3-tier approval: initial → site inspection → final, with state machine
- 终审通过自动入库，变更 supplier 状态为"已准入" / Auto-onboarding, status change to "admitted"

**2.2.2 绩效评估 / Performance Evaluation**

- 灵活评估模板：季度/年度/专项评估，自定义评分维度与权重 / Flexible templates: quarterly/annual/special, custom dimensions & weights
- 多维度打分：质量/交付/成本/服务，自动计算总分并定级（A/B/C/D） / Multi-dimension: quality/delivery/cost/service, auto grading (A/B/C/D)
- 供应商确认与整改跟踪闭环，评估结果同步更新供应商评级 / Confirmation & rectification loop, rating sync

**2.2.3 供需协同 / Supply-Demand Collaboration**

- 预测计划发布：向供应商推送物料需求预测（含置信度分级 HIGH/MEDIUM/LOW） / Forecast publishing with confidence grading
- 发货物流追踪：发货单 → 运输中 → 到货 → 签收，全状态追踪 / Delivery tracking: shipped → in transit → arrived → signed
- 对账结算闭环：生成对账单 → 双方确认 → 开票 → 付款 / Reconciliation: generate → mutual confirm → invoice → payment

**2.2.4 风险管理 / Risk Management**

- 风险事件分级处理：法律/财务/质量/交期/信用五大类型，LOW~CRITICAL 四级 / 5 risk types (legal/financial/quality/delivery/credit), 4 severity levels
- 智能预警引擎：资质到期/交期延迟/质量缺陷/评分下滑，自动生成预警记录 / Smart alerts: cert expiry/delay/defect/score decline
- 动态风险监控：对接征信/舆情/内部来源（`pullExternalRisk()`） / Dynamic monitoring: credit/public opinion/internal sources
- 黑名单管理：永久/临时拉黑，采购/招标/询价操作前置黑名单拦截校验 / Blacklist: permanent/temporary, pre-check on purchase/bidding/inquiry

**2.2.5 配额管理 / Quota Management (v1.2.10)**

- 按供应商评级实行差异化配额分配 / Differentiated quota allocation by supplier grade
- 同物料品类下所有供应商配额总和 ≤ 100% 校验 / Same-category sum ≤ 100% validation
- 按评级/绩效自动计算配额权重 / Auto-calculate quota weight by grade & performance

**2.2.6 整改跟踪 / Rectification Tracking (v1.2.10)**

- 问题类型：QUALITY/DELIVERY/SERVICE/CERT，MAJOR/MINOR/CRITICAL 严重程度 / Issue types & severity levels
- 整改生命周期：待整改 → 方案已提交 → 整改中 → 待复评 → 已完成/逾期 / Lifecycle: pending → plan submitted → in progress → review → done/overdue
- 8D 报告跟踪：整改方案 + 证据附件 + 复评关联 / 8D report: plan + evidence + review linkage

**2.2.7 财务结算协同 / Financial Settlement (v1.2.10)**

- 结算单生成（结算单号 + 周期 + 金额） / Settlement bill generation (bill number + period + amount)
- 双方确认流转：供应商确认 → 采购方确认 → 已结算 / Mutual confirmation: supplier → buyer → settled
- 三单匹配：订单/入库单/发票自动对账（±0.01 容差自动通过） / 3-way matching: PO/receipt/invoice (±0.01 tolerance)

**2.2.8 需求预测协同 / Demand Forecast Collaboration (v1.2.10)**

- 需求预测录入（置信度分级 + 来源标注 SALES/PLAN/HISTORY） / Forecast entry with confidence & source
- 分享供应商 → 供应商反馈承诺量 / Share to suppliers → feedback commitment

### 2.3 物料管理 / Material Management

物料管理覆盖四大功能域，扩展自 goods 主数据表： / Covers 4 functional domains, extending from goods master data:

**2.3.1 基础数据管理 / Basic Data**

- 物料主数据：编码/名称/规格/品牌/单位/物料类型（UNIQUE/BATCH） / Material master: code/name/spec/brand/unit/type
- 分类树：多级树形分类（parent_id），支持编码唯一约束 / Category tree: multi-level, unique code constraint
- 多单位换算：基本单位与采购/库存/发料单位之间换算率管理 / Multi-unit conversion rate management
- 安全库存、最小订货量、采购提前期等供应链参数 / Safety stock, MOQ, lead time parameters

**2.3.2 需求与计划 / Demand & Planning**

- BOM 管理：物料清单版本管理、发布审批、成本预估 / BOM: version control, release approval, cost estimation
- MRP 净需求计算：基于 BOM + 库存 + 在途量计算净需求 / MRP net requirement: BOM + inventory + in-transit

**2.3.3 采购执行协同 / Procurement Execution Collaboration**

- 供应商发货协同：发货单创建/发货/收货，物流跟踪 / Delivery collaboration: create/ship/receive, tracking
- 生产进度填报：供应商按采购订单填报生产进度 / Production progress: per-PO progress reporting
- 来料检验 IQC：检验单创建/检验/合格/缺陷判定 / IQC: inspection creation/inspection/pass/defect

**2.3.4 质量追溯与分析 / Quality Traceability & Analysis**

- 全链路批次追溯：按批次号/条码追溯物料流转路径 / Full-chain batch tracing: by batch/barcode
- 供应商绩效评级：按物料维度统计供应商交付/质量表现 / Supplier performance rating by material
- 物料周转分析：按周期统计物料采购金额/库存金额/成本趋势 / Material turnover analysis: purchase/inventory/cost trends

### 2.4 合同管理 / Contract Management

- 合同生命周期管理：从草稿到完成的 10 个状态完整流转 / Full lifecycle: 10 states from draft to complete
- 状态机守卫：`ContractStatusEnum` 枚举内置 `allowTargets` 合法跳转校验 / State machine guards with allowTargets
- 审批流：对接 Flowable 7.0 工作流引擎 / Approval via Flowable 7.0
- 变更日志追溯：每次变更记录 before/after JSON 快照 / Change log: before/after JSON snapshots
- 电子签章：签章字段扩展 + 风险条款预警（`contract_risk_clause`） / E-signature + risk clause alerts

### 2.5 采购管理 / Procurement Management

**核心采购订单 / Core Purchase Order**

- 采购订单创建（关联合同与供应商）、幂等校验（`uk_request_id`） / PO creation with idempotency check
- 提交订单触发库存扣减（乐观锁 + 自旋重试） / Submit triggers inventory deduction (optimistic lock + retry)
- 状态流转：草稿 → 待审批 → 已审批 → 执行中 → 已完成/已取消 / State flow: draft → pending → approved → executing → done/cancelled

**9 种采购模式 / 9 Procurement Modes**

| 序号 / # | 采购模式 / Mode | 说明 / Description |
|------|----------|------|
| 1 | 公开招标 / Open Bidding | 发布 → 投标 → 开标 → 评标 → 定标 → 流标/终止 / Publish → bid → open → evaluate → award → cancel |
| 2 | 邀请招标 / Invited Bidding | 邀请特定供应商 → 投标 → 开标 → 评标 → 定标 / Invite specific suppliers → bid → evaluate → award |
| 3 | 询比采购 / Inquiry | 发布询价 → 供应商报价 → 结果确认 / Publish inquiry → quotes → confirm |
| 4 | 竞价采购 / Auction | 竞价场次（英式/荷兰式/日式）→ 出价记录 → 结果确认 / Auction session → bids → confirm |
| 5 | 竞争性谈判 / Competitive Negotiation | 谈判会话 → 多轮报价 → 综合评审 → 最终定价 / Sessions → multi-round → evaluation → final price |
| 6 | 竞争性磋商 / Competitive Consultation | 磋商会话 → 提交响应文件 → 综合评审 → 确定成交 / Sessions → submissions → evaluation → award |
| 7 | 单一来源 / Single Source | 直接指定供应商采购 / Direct procurement from designated supplier |
| 8 | 框架协议 / Framework Agreement | 协议签订 → 供应商入围 → 二次竞价/直接下单 / Agreement → shortlist → secondary auction / direct order |
| 9 | 合作创新采购 / Cooperative Innovation | 研发合作 → 首购 → 推广应用 / R&D cooperation → first purchase → promotion |

**竞价大厅 / Bidding Hall (v1.2.10)**：英式/荷兰式/日式三种竞价方式、实时排名刷新 <0.05s、身份隐藏公开排名、自动延时 / Three auction types, real-time ranking <0.05s, identity hidden, auto-extension

**价格库 / Price Library (v1.2.10)**：合同价/报价/现货/协议价四种类型、有效期管理、月度走势分析、自动比价 / Four price types, validity management, trend analysis, auto comparison

**供应商推荐 / Supplier Recommendation (v1.2.10)**：历史交易/同类品类/同区域/资质匹配四维度评分 / 4-dimension scoring

### 2.6 库存管理 / Inventory Management

- 出/入库操作，库存变更日志全量记录（before/after 快照） / Inbound/outbound with full change logging
- 安全库存阈值告警（低库存实时预警） / Safety stock threshold alert
- 乐观锁并发控制（`version` 字段 + 自旋重试） / Optimistic lock concurrency control
- Seata AT 模式分布式事务预留（`undo_log` 表已就位） / Seata AT distributed transaction (undo_log ready)

### 2.7 收货管理 / Receipt Management

- 收货单创建 → 质检 → 确认入库 / Receipt creation → QC → confirm stock-in
- 消息表补偿：确认收货写入 `receipt_outbox`，定时扫描补发（最多 5 次） / Outbox compensation: write + scheduled retry (max 5)
- 采购-库存联动：RocketMQ 异步入库通知 inventory / Purchase-inventory linkage via RocketMQ

### 2.8 工作流引擎 / Workflow Engine

- Flowable 7.0 集成：流程定义/启动/审批/待办查询/审批历史追溯 / Process definition, start, approval, pending tasks, history
- 支持合同审批、采购审批等多种业务流程 / Supports contract approval, purchase approval, etc.

### 2.9 开放平台 / Open Platform (v1.2.10)

- **API 鉴权 / API Auth**：AppKey/Secret 自动生成 + HMAC-SHA256 签名验证 + 防重放 ±5min / Auto-generated keys + HMAC-SHA256 + anti-replay
- **Webhook**：ORDER_CREATED / ORDER_STATUS_CHANGE / RECEIPT_CONFIRMED / INVOICE_UPLOADED 事件推送 + 3 次重试 / Event push with 3 retries
- **动态 API 对接引擎 / Dynamic API Engine**：6 种鉴权方式 + `{{变量}}` 模板 + JsonPath 响应映射 + 全量调用日志 / 6 auth methods + template + JsonPath + full logging
- **标准连接器 / Standard Connectors**：OrderConnector / ReceiptConnector / SupplierConnector / MaterialConnector
