## 二、Atlas 现状盘点 / Atlas Current Status

Atlas 是一个基于 Spring Cloud 微服务架构的 SRM（Supplier Relationship Management）系统，目前包含 12 个核心微服务模块（v1.2.21 新增 atlas-message 和 atlas-supplier-portal 子包）：

Atlas is a microservices-based SRM (Supplier Relationship Management) system built on Spring Cloud, currently comprising 12 core microservice modules (v1.2.21 added atlas-message and atlas-supplier-portal sub-package):

| 模块 / Module | 功能描述 / Function Description |
|---|---|
| **atlas-common** | 公共组件、通用工具类、国际化（i18n）基础支持、WebSocket 基础设施 / Common components, utilities, i18n foundation, WebSocket infrastructure |
| **atlas-user** | 用户认证与权限管理 / User authentication and authorization management |
| **atlas-supplier** | 供应商管理（含评估/风险/配额/交付/BOM/MRP/IQC/物料/产量进度/供应商门户子包）/ Supplier management (including assessment, risk, quota, delivery, BOM, MRP, IQC, materials, production progress, supplier portal sub-package) |
| **atlas-contract** | 合同管理 + 电子合同子包（模板/签署流程/条款比对/履约跟踪）/ Contract management + e-contract sub-package (template/signing flow/clause comparison/performance tracking) |
| **atlas-purchase** | 采购管理（订单/招标/询价/竞价/拍卖/单一来源）/ Purchase management (PO, bidding, RFQ, auction, single source) |
| **atlas-inventory** | 库存管理 / Inventory management |
| **atlas-receipt** | 收货管理 / Receipt management |
| **atlas-open** | API 集成与开放接口（DynamicApiService）/ API integration and open interface (DynamicApiService) |
| **atlas-workflow** | 工作流引擎（基于 Flowable）/ Workflow engine (Flowable-based) |
| **atlas-gateway** | API 网关 / API gateway |
| **atlas-message** | 消息中心（WebSocket实时推送 + 邮件 + 短信三通道）/ Message center (WebSocket real-time push + email + SMS triple-channel) |

## 九、P3 — 智能化演进 / Intelligent Evolution

> **周期 / Timeline**：6-12 个月 / 6-12 months
> **目标 / Objective**：引入 AI 大模型和高级风控能力，构建智能采购护城河 / Introduce AI large models and advanced risk control to build intelligent procurement moat

### 9.1 AI 采购助手 / AI Procurement Assistant

**差距 / Gap**：Atlas 完全没有 AI 辅助能力。Atlas has no AI assistance capability.

**建议方案 / Recommendation**：
- **寻源推荐 / Sourcing Recommendation**：基于历史采购数据和供应商绩效，大模型辅助推荐最优供应商和寻源策略 / LLM-assisted optimal supplier and sourcing strategy recommendation based on historical data
- **合同风险审查 / Contract Risk Review**：大模型自动审查合同条款，识别风险项并给出修改建议 / LLM auto-review of contract clauses, identifying risks and suggesting modifications
- **采购决策建议 / Procurement Decision Suggestions**：基于价格趋势、市场行情、库存水位等多维数据生成采购建议 / Generate procurement suggestions based on price trends, market conditions, and inventory levels
- **自然语言查询 / Natural Language Query**：采购员可通过自然语言查询采购数据（如"上个月A类物料哪个供应商价格最低"）/ Natural language query of procurement data

**技术路径 / Technical Path**：
```
新增模块 atlas-ai
大模型接入：混元/DeepSeek API（可复用现有 AI 基础设施）
RAG 架构：采购数据向量化 + 检索增强生成
Prompt 工程：针对采购场景的专用 Prompt 模板
```

**预期收益 / Expected Benefits**：
- 寻源决策效率提升 40%+ / Sourcing decision efficiency improved by 40%+
- 合同风险遗漏率降低 60%+ / Contract risk omission rate reduced by 60%+

### 9.2 围标预警 / Collusion Detection

**差距 / Gap**：Atlas 无任何围标检测机制。Atlas has no collusion detection mechanism.

**建议方案 / Recommendation**：
- IP 地址分析：对比投标供应商的访问 IP，识别相同 IP 投标 / IP analysis: compare bidder access IPs, identify same-IP bidding
- 标书相似度检测：基于文本相似度算法（TF-IDF/Cosine Similarity）检测标书雷同 / Bid similarity detection: TF-IDF/Cosine Similarity for bid content duplication
- 报价模式分析：检测异常报价模式（如围标常见的阶梯报价、陪标报价）/ Pricing pattern analysis: detect anomalous patterns (step pricing, companion bidding)
- 关联关系图谱：基于工商数据构建供应商关联关系图谱 / Relationship graph: build supplier relationship graph based on business registration data

**技术路径 / Technical Path**：
```
新增模块 atlas-anti-fraud
IP 记录：atlas-gateway 拦截器记录投标请求 IP
文本相似度：Apache Lucene / 自研 TF-IDF
图谱分析：Neo4j / 自研图算法
```

**预期收益 / Expected Benefits**：
- 围标行为发现率提升 80%+ / Collusion detection rate improved by 80%+
- 采购公平性保障 / Procurement fairness assurance

### 9.3 合规认证体系 / Compliance Certification System

**差距 / Gap**：Atlas 无任何信息安全合规认证。Atlas has no information security compliance certification.

**建议方案 / Recommendation**：
- 启动 ISO27001 信息安全管理体系建设 / Initiate ISO27001 information security management system
- 推进等保三级（网络安全等级保护 2.0）认证 / Advance China Classified Protection Level 3 (CSL 2.0) certification
- 数据安全加固：数据加密（传输 TLS 1.3 + 存储 AES-256）、访问控制增强、审计日志完善 / Data security hardening: encryption (TLS 1.3 in transit + AES-256 at rest), access control enhancement, audit log completeness
- 隐私合规：参照 GDPR 要求完善用户数据管理流程 / Privacy compliance: improve user data management per GDPR requirements

**技术路径 / Technical Path**：
```
安全基础设施加固
- 传输层：全站 HTTPS + TLS 1.3
- 存储层：敏感字段 AES-256 加密
- 审计层：操作日志全量记录 + 不可篡改存储
- 访问控制：RBAC 增强 + 数据脱敏
合规文档体系建设 + 第三方审计
```

**预期收益 / Expected Benefits**：
- 满足政企客户安全准入要求 / Meet government and enterprise security requirements
- 为进入海外市场提供合规基础 / Provide compliance foundation for overseas markets
